# CSDN 风格本地 TXT 阅读器

> 一个高度仿 CSDN 博客页面的**本地 TXT 阅读器**。
> **纯前端 · 零依赖 · 双击 `index.html` 就能用**。

把长篇小说、手册、笔记这类 txt 塞进来，就能获得类似在 CSDN 上读技术长文的阅读体验：
左边作者卡片/热门文章、中间正文、右边章节目录 & 相关推荐，还能搜章节、切章节、GBK 自动解码。

## ✨ 特性

- **CSDN 像素级仿真**：顶部导航、左右侧栏、正文卡片、颜色/字号/间距全部对齐 CSDN 博客页
- **纯前端，零依赖**：不需要 Node.js、不需要构建、不联网也能用
- **智能章节切分**：识别中文格式 `第X章 / 第X节 / 第X回 / 第X卷 / 第X篇 / 第X集 / 第X部`
- **单章展示**：一次只显示一章，读长小说不卡；上一章 / 下一章按钮直接翻页
- **编码自动检测**：UTF-8 / GBK / GB18030 全部支持，GBK 中文小说不会乱码
- **右栏两种模式**：目录模式（章节列表 + 搜索）↔ CSDN 卡片模式（客户端/相关推荐/学习路线/登录）
- **拖拽上传**：把 txt 拖到中间正文区就行，也可以点「选择文件」

## 🚀 快速开始

### 方式 1：双击运行（最简单）

```bash
# 1) 克隆或下载本项目
git clone <你的仓库地址>
cd csdn-txt-reader

# 2) 双击 index.html，用浏览器打开即可
```

推荐浏览器：**Chrome / Edge / Safari 最新版**。macOS 上直接 `open index.html`，Windows 双击即可。

### 方式 2：本地启一个静态服务器（可选）

某些浏览器对 `file://` 有严格的安全限制。如果双击打开异常，用任意静态服务器都行：

```bash
# Python 3
python3 -m http.server 8000

# Node.js
npx serve .

# 然后浏览器打开 http://localhost:8000
```

### 使用步骤

1. 打开页面
2. 把你的 `.txt` 文件**拖到中间正文区**，或点「选择文件」
3. 页面自动解析章节，右侧目录列出全部章节
4. 点击目录任意章节 → 中间只显示该章
5. 用「上一章 / 下一章」按钮翻页
6. 目录搜索框可实时过滤章节标题
7. 目录卡片右上角 `⟩` 按钮可收起目录，切换到 CSDN 风格右侧栏

## 📖 支持的章节格式

解析器基于正则识别，标题需**顶格**（无全角空格 `　` 缩进），格式如下：

| 支持 | 示例 |
|------|------|
| ✅ 中文数字 + 章 | `第一章 旧土` `第十二章 惊变` |
| ✅ 阿拉伯数字 + 章 | `第 1 章 旧土` `第123章 危机出现` |
| ✅ 章/节/回/卷/篇/集/部 | `第三节 XX` `第五回 XX` `第二卷 XX` |
| ✅ 章节后带任意分隔符 | `第一章?旧土`（`?`/全角空格/多个空格） |
| ❌ 英文 Chapter | `Chapter 1` (未支持) |
| ❌ 缩进的章节 | `　　第一章 XX`（会被当作正文段落） |

如果你的 txt 章节格式不同，改一下 [js/parser.js](js/parser.js) 里的 `CHAPTER_RE` 正则就行。

## 🧩 自定义

### 换文章标题 / 作者信息

编辑 [index.html](index.html)：
- 顶部 `<title>` 标签
- `.article-title` 里的文章标题
- `.author-card` 里的头像、用户名、粉丝数等
- `.article-meta` 里的日期、阅读量、点赞数

### 换配色

主品牌色是 CSDN 红 `#fc5531`。全局替换即可换主题色：

```bash
# 例：换成蓝色
sed -i '' 's/#fc5531/#1a73e8/g' css/style.css
```

### 修改章节识别规则

打开 [js/parser.js](js/parser.js)，改 `CHAPTER_RE` 正则。例如支持 `Chapter X`：

```javascript
var CHAPTER_RE = /^[ \t\u00a0]*(第\s*[0-9零一二三四五六七八九十百千两]+\s*[章节回卷篇集部]|Chapter\s+\d+).*$/i;
```

### 右侧卡片内容（相关推荐 / 学习路线）

编辑 [index.html](index.html) 中的 `#rightCsdn` 块，直接改文案即可。

## ❓ 常见问题

**Q: 上传后中文乱码？**
A: 检查文件编码。项目自动尝试 UTF-8 → GBK → GB18030，理论上主流中文编码都能识别。如果还乱码，用 VSCode / EmEditor 另存为 UTF-8 后再上传。

**Q: 识别出很多"假章节"？**
A: 通常是正文里以「第X章」开头的段落被误判。检查该行是否有全角空格缩进（`　　`）——正常段落有缩进就会被过滤掉。如果没有缩进，可以人工把标题格式与正文区分开。

**Q: 上传特别大的 txt（10 MB+）会卡吗？**
A: 亲测 11.5 MB（近 10 万行、677 章的《深空彼岸.txt》）秒开无压力。因为只在切章时渲染当前一章的段落。

**Q: 双击打开一片空白？**
A: 少数浏览器（如某些版本 Firefox）对 `file://` 下的 JS 执行有限制。用「方式 2」起个本地服务器即可。

**Q: 想部署到网上给别人访问？**
A: 直接把整个目录扔到任何静态托管（GitHub Pages / Vercel / Netlify / OSS）即可，不需要构建。

## 🛠 技术栈

| 组件 | 用途 |
|------|------|
| 原生 HTML + CSS + JS | 页面结构 / 样式 / 交互 |
| CSS Grid | 三栏布局（`grid-template-rows: minmax(0, 1fr)` 是滚动关键） |
| FileReader API | 读取用户上传的 txt 为 ArrayBuffer |
| TextDecoder | UTF-8 / GBK / GB18030 解码 |

无框架、无打包工具、无 npm 依赖。

## 📝 已知限制

- 章节识别只支持**中文**格式的标记（`第X章` 系列）
- 目录只支持**单级**（不做「卷 > 章 > 节」多级嵌套）
- 若源 txt 内容重复（如文件把前 N 章又贴了一遍），会得到重复的目录项，属正常
- 界面视觉参考 CSDN 博客页，非官方产品，仅用于本地学习/阅读
